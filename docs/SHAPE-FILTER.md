# Riverbraid Shape Filter - Structural Artifact Admissibility
RDK v1.5.0 Companion Specification

## I. The Core Reframe
Riverbraid does not interpret. It measures. [cite: 9] This filter rejects any artifact whose structure cannot be traced, verified, bounded, or attributed without identity distortion. [cite: 113, 123] Rejection is a measurement, not a judgment. [cite: 157]

## II. The Shape Envelope
Every admissible artifact must satisfy these four dimensions: [cite: 11, 27]
1. **PROVENANCE TRACEABLE**: Verifiable source chain. [cite: 15]
2. **TRANSFORMATIONS DECLARED**: Recorded modification ledger. [cite: 18]
3. **IDENTITY PRESERVED**: Bounded real identities without synthetic collapse. [cite: 21, 22]
4. **ENTROPY BOUNDED**: Generative distortion within declared upper bounds. [cite: 25]

## III. The Shape Gate Execution Sequence
The gate runs in strict order; the first failure terminates the check: 
1. **PROVENANCE CHECK**: Reconstruct the source chain. [cite: 74, 75]
2. **TRANSFORMATION CHECK**: Verify ledger entries against content hash. [cite: 78, 82]
3. **IDENTITY CHECK**: Detect Identity Collapse or unauthorized synthetic output. [cite: 84, 88]
4. **AUTHORIZATION CHECK**: Verify valid records for protected sources. [cite: 89, 91]
5. **ENTROPY CHECK**: Measure distortion against the declared bound. [cite: 93, 95]

## IV. Normative Constraints (Section X)
The shape filter MUST NEVER be extended to include semantic interpretation, category-based prohibitions, or jurisdiction-specific rules. [cite: 138, 140, 143] If the filter begins to decide rather than measure, it has left Riverbraid's scope. [cite: 145]
