## 2. Primary Claim
Riverbraid is a functional, inspectable starting point for AI governance, transparency, and trust. It establishes a deterministic verification floor that separates what is proven from what is experimental, local, incomplete, or not yet externally reproduced.
## 3. Included Proof Boundaries
1. Phase 24 Functional Registry: 30 repositories verified to have reached a passing cluster state.
2. Phase 25C Repair Plan: Structured tracking of outer repository repair vectors.
3. Phase 25E Local Outer Surface: Post alignment verification showing 23 clean, balanced outer surfaces.
4. Phase 26 Comprehension Surface: Public markdown baseline explaining claim levels, boundary maps, and inspection guides.
## 4. Explicit Excluded Claims
1. No External Reproduction Claim: System results have not been verified by independent external environments.
2. No Third Party Certification Claim: No institutional or regulatory body has evaluated or endorsed this baseline.
3. No Absolute Security Claim: The verification floor checks architectural posture, not dynamic runtime vulnerability immunity.
4. No Absence of Defects Claim: The baseline guarantees layout and state alignment, not error free code execution.
5. No Complete Governance Institution Claim: Intended strictly as an inspectable technical floor, not an operational governance framework.
6. No Organization Wide Registry Inheritance: Outer repositories do not inherit the functional registry status of Core or the Phase 24 cohort.
7. No Domain Specific Logic Maturity Claim: Presence within the boundary proves layout structural hygiene, not comprehensive domain maturity.
