# Contributing

To maintain the integrity of this repository, all contributions must follow the strict verification path.

## Local Development
1. Fork the repository.
2. Make your changes.
3. Run the verifier locally to ensure you haven't broken the manifest.
4. Update the manifest if your changes are intentional.

## Standards
* No dependencies: Keep the core logic in plain, deterministic scripts.
* No "AI-speak": Write documentation that describes the mechanism, not the vision.
* Fail-Closed: If a test can fail, it must stop the build.
